# MongoDB Guide for InvitaWedds

This guide covers installing, starting, connecting to, inspecting, querying, backing up, and troubleshooting MongoDB for the InvitaWedds application.

The application uses:

| Setting | Value |
|---|---|
| Database | `invitavideodb` |
| Main collection | `renders` |
| Local address | `127.0.0.1:27017` |
| Backend driver | Motor/PyMongo |
| Backend storage selector | `STORAGE_BACKEND=mongodb` |

MongoDB terminology differs from SQL:

| SQL term | MongoDB term |
|---|---|
| Database | Database |
| Table | Collection |
| Row/record | Document |
| Column | Field |
| Primary key | `_id` |

## 1. Local macOS installation

Install MongoDB Community using Homebrew:

```bash
brew tap mongodb/brew
brew trust --formula mongodb/brew/mongodb-community@8.0
brew update
brew install mongodb/brew/mongodb-community@8.0
```

Start it as a background service:

```bash
brew services start mongodb-community@8.0
```

Check the service and server:

```bash
brew services list
mongosh --eval 'db.runCommand({ ping: 1 })'
```

The ping result should contain `ok: 1`.

Service commands:

```bash
brew services start mongodb-community@8.0
brew services stop mongodb-community@8.0
brew services restart mongodb-community@8.0
```

## 2. Configure the local backend

Set the following values in `backend/.env`:

```dotenv
STORAGE_BACKEND=mongodb
MONGO_URL=mongodb://127.0.0.1:27017
DB_NAME=invitavideodb
```

Restart the backend after changing `.env`. Environment changes are not loaded by an already-running process.

```bash
cd /Volumes/Projects/WeddingVideoApp/backend
source .venv/bin/activate

GOOGLE_CLIENT_ID="YOUR_GOOGLE_CLIENT_ID" \
python -m uvicorn server:app --host 127.0.0.1 --port 8001
```

Verify that the backend selected MongoDB:

```bash
curl -fsS http://127.0.0.1:8001/api/health
```

The response should include:

```json
"storage_backend": "mongodb"
```

## 3. Create the application database

MongoDB creates a database only after a collection or document is created. Running `use invitavideodb` alone does not persist an empty database.

Open the shell:

```bash
mongosh
```

Then run:

```javascript
use invitavideodb

db.createCollection("renders")
db.renders.createIndex({ created_at: -1 })
db.renders.createIndex({ status: 1, created_at: -1 })
```

Exit with:

```javascript
exit
```

The application also creates `renders` automatically when its first render job is submitted.

## 4. Connect or log in

### Local development without authentication

```bash
mongosh mongodb://127.0.0.1:27017/invitavideodb
```

Or simply:

```bash
mongosh
```

Then select the database:

```javascript
use invitavideodb
```

### Production with authentication

Use a password prompt so the password is not saved in shell history:

```bash
mongosh \
  --host 127.0.0.1 \
  --port 27017 \
  --username invitavideo_app \
  --authenticationDatabase invitavideodb \
  --password \
  invitavideodb
```

Enter the database password when prompted.

Do not expose port `27017` publicly. On the VPS, connect over SSH and run `mongosh` locally on the server.

## 5. Navigate databases and collections

Inside `mongosh`:

```javascript
// Show databases that currently contain data.
show dbs

// Select the application database.
use invitavideodb

// Show the current database name.
db.getName()

// Show collections (equivalent to SQL tables).
show collections

// Return collection names as an array.
db.getCollectionNames()

// Show collection statistics.
db.renders.stats()

// Show database statistics.
db.stats()
```

One-line terminal equivalents:

```bash
mongosh --quiet --eval 'db.adminCommand({ listDatabases: 1, nameOnly: true })'
mongosh --quiet --eval 'db.getSiblingDB("invitavideodb").getCollectionNames()'
mongosh --quiet --eval 'db.getSiblingDB("invitavideodb").renders.countDocuments()'
```

## 6. Understand a render document

The backend stores render-job metadata in `invitavideodb.renders`. A typical document resembles:

```javascript
{
  _id: "4994de68b10e41b6a1860384d0d81dd7",
  template: "marigold",
  couple: {
    partnerOne: "Aisha",
    partnerTwo: "Rohan"
  },
  eventDate: "November 21, 2026",
  venue: {
    name: "The Leela Palace",
    city: "Udaipur"
  },
  displayMessage: "Wedding invitation message",
  durationInSeconds: 30,
  musicId: "tere-sang",
  status: "done",
  progress: 1,
  created_at: "2026-07-13T10:00:00+00:00",
  finished_at: "2026-07-13T10:01:00+00:00",
  size_bytes: 12345678
}
```

The `_id` is a string generated by the application, not a MongoDB `ObjectId`.

Possible statuses include `queued`, `rendering`, `done`, and `failed`.

MongoDB stores only job metadata. Uploaded images and rendered MP4 files are stored on disk under `backend/uploads` and `backend/renders` locally.

## 7. Read and search records

Select the database first:

```javascript
use invitavideodb
```

Count every record:

```javascript
db.renders.countDocuments()
```

Show one record:

```javascript
db.renders.findOne()
```

Show every record:

```javascript
db.renders.find()
```

Show the latest 10:

```javascript
db.renders.find().sort({ created_at: -1 }).limit(10)
```

Find one render by its application ID:

```javascript
db.renders.findOne({ _id: "YOUR_RENDER_ID" })
```

Find completed jobs:

```javascript
db.renders.find({ status: "done" }).sort({ created_at: -1 })
```

Find failed jobs and show their errors:

```javascript
db.renders.find(
  { status: "failed" },
  { _id: 1, created_at: 1, error: 1 }
).sort({ created_at: -1 })
```

Find active jobs:

```javascript
db.renders.find({ status: { $in: ["queued", "rendering"] } })
```

Search nested fields:

```javascript
db.renders.find({ "couple.partnerOne": "Aisha" })
db.renders.find({ "venue.city": "Udaipur" })
```

Search text using a case-insensitive regular expression:

```javascript
db.renders.find({
  "couple.partnerOne": { $regex: "aisha", $options: "i" }
})
```

Show only selected fields:

```javascript
db.renders.find(
  {},
  {
    _id: 1,
    status: 1,
    couple: 1,
    eventDate: 1,
    created_at: 1
  }
).sort({ created_at: -1 })
```

Count records by status:

```javascript
db.renders.aggregate([
  {
    $group: {
      _id: "$status",
      count: { $sum: 1 }
    }
  },
  { $sort: { count: -1 } }
])
```

Calculate total stored video size recorded in MongoDB:

```javascript
db.renders.aggregate([
  {
    $group: {
      _id: null,
      totalBytes: { $sum: { $ifNull: ["$size_bytes", 0] } }
    }
  }
])
```

## 8. Insert and update records

Normally, create render records through the application API rather than manually.

Insert a clearly marked test document:

```javascript
db.renders.insertOne({
  _id: "manual-test-record",
  template: "marigold",
  couple: {
    partnerOne: "Test One",
    partnerTwo: "Test Two"
  },
  status: "test",
  progress: 0,
  created_at: new Date().toISOString()
})
```

Update one record:

```javascript
db.renders.updateOne(
  { _id: "manual-test-record" },
  { $set: { status: "done", progress: 1 } }
)
```

Add a field:

```javascript
db.renders.updateOne(
  { _id: "manual-test-record" },
  { $set: { note: "Created manually for testing" } }
)
```

Remove a field:

```javascript
db.renders.updateOne(
  { _id: "manual-test-record" },
  { $unset: { note: "" } }
)
```

Read a document before and after changing it. Manual updates can make MongoDB metadata disagree with files on disk or the render service.

## 9. Delete records safely

Preview the exact record first:

```javascript
db.renders.findOne({ _id: "manual-test-record" })
```

Delete that one record:

```javascript
db.renders.deleteOne({ _id: "manual-test-record" })
```

Preview failed records before deleting any:

```javascript
db.renders.find(
  { status: "failed" },
  { _id: 1, created_at: 1, error: 1 }
)
```

Avoid `deleteMany({})`, `db.renders.drop()`, and `db.dropDatabase()` unless you intend to remove all application metadata. Deleting MongoDB records does not delete related uploaded images or MP4 files from disk.

## 10. Inspect and manage indexes

List indexes:

```javascript
db.renders.getIndexes()
```

Create the recommended indexes:

```javascript
db.renders.createIndex({ created_at: -1 })
db.renders.createIndex({ status: 1, created_at: -1 })
```

Inspect how MongoDB executes a query:

```javascript
db.renders.find({ status: "done" })
  .sort({ created_at: -1 })
  .explain("executionStats")
```

## 11. View records through the application API

The backend provides read endpoints for render records:

```bash
# Latest render records
curl -fsS http://127.0.0.1:8001/api/renders

# One render record
curl -fsS http://127.0.0.1:8001/api/renders/YOUR_RENDER_ID
```

If `jq` is installed, format the output:

```bash
curl -fsS http://127.0.0.1:8001/api/renders | jq
```

The API returns a selected subset of fields; `mongosh` shows the complete stored document.

## 12. Create an authenticated production user

Local development can use MongoDB without authentication when it is bound only to loopback. A production VPS should enable authentication and remain bound to `127.0.0.1`.

Before enabling authorization, connect as the initial local administrator and create users. Use `passwordPrompt()` to avoid writing passwords into shell history:

```bash
mongosh
```

```javascript
use admin

db.createUser({
  user: "admin",
  pwd: passwordPrompt(),
  roles: ["root"]
})

use invitavideodb

db.createUser({
  user: "invitavideo_app",
  pwd: passwordPrompt(),
  roles: [
    { role: "readWrite", db: "invitavideodb" }
  ]
})
```

Configure `/etc/mongod.conf` on the VPS:

```yaml
net:
  bindIp: 127.0.0.1

security:
  authorization: enabled
```

Restart MongoDB:

```bash
sudo systemctl restart mongod
sudo systemctl status mongod --no-pager
```

Use a percent-encoded password in the backend connection string if it contains characters such as `@`, `:`, `/`, `?`, or `#`:

```dotenv
STORAGE_BACKEND=mongodb
MONGO_URL=mongodb://invitavideo_app:ENCODED_PASSWORD@127.0.0.1:27017/invitavideodb?authSource=invitavideodb
DB_NAME=invitavideodb
```

Protect production environment files:

```bash
sudo chown root:invitawedds /etc/invitawedds/backend.env
sudo chmod 640 /etc/invitawedds/backend.env
```

## 13. Back up and restore

### Local backup

Create a compressed archive:

```bash
mkdir -p "$HOME/mongodb-backups"

mongodump \
  --uri="mongodb://127.0.0.1:27017/invitavideodb" \
  --archive="$HOME/mongodb-backups/invitavideodb.archive.gz" \
  --gzip
```

Restore it:

```bash
mongorestore \
  --uri="mongodb://127.0.0.1:27017/invitavideodb" \
  --archive="$HOME/mongodb-backups/invitavideodb.archive.gz" \
  --gzip
```

Add `--drop` only when you intentionally want the restore to replace existing collections.

### Production backup with a password prompt

To avoid putting a password in shell history, use separate options and allow the tool to prompt:

```bash
mongodump \
  --host=127.0.0.1 \
  --port=27017 \
  --username=invitavideo_app \
  --authenticationDatabase=invitavideodb \
  --db=invitavideodb \
  --archive=/var/backups/invitavideodb/invitavideodb.archive.gz \
  --gzip
```

## 14. Troubleshooting

### `mongosh: command not found`

MongoDB Shell is not installed or is not on `PATH`.

On macOS:

```bash
brew install mongodb/brew/mongodb-community@8.0
```

On an Ubuntu VPS with the MongoDB repository configured:

```bash
sudo apt update
sudo apt install -y mongodb-mongosh
```

### Connection refused on port 27017

macOS:

```bash
brew services list
brew services restart mongodb-community@8.0
```

Ubuntu:

```bash
sudo systemctl status mongod --no-pager
sudo journalctl -u mongod -n 100 --no-pager
sudo systemctl restart mongod
```

### `getCollectionNames()` returns `[]`

The database is reachable but contains no collections. Create one or submit the first render job:

```javascript
use invitavideodb
db.createCollection("renders")
```

### Backend health says `storage_backend: memory`

Confirm `backend/.env` contains:

```dotenv
STORAGE_BACKEND=mongodb
```

Then completely restart the backend. An already-running process will not reload `.env` automatically.

### Authentication failed

Check all of the following:

- The username is `invitavideo_app`.
- The user was created in the `invitavideodb` database.
- The URI includes `authSource=invitavideodb`.
- Special characters in a URI password are percent-encoded.
- The backend was restarted after changing its environment file.

Test interactively without exposing the password:

```bash
mongosh \
  --host 127.0.0.1 \
  --username invitavideo_app \
  --authenticationDatabase invitavideodb \
  --password \
  invitavideodb
```

### Database is not shown by `show dbs`

MongoDB does not display an empty database. Create a collection or insert a document first.

## 15. Daily command cheat sheet

```bash
# Start local MongoDB
brew services start mongodb-community@8.0

# Connect directly to the app database
mongosh mongodb://127.0.0.1:27017/invitavideodb

# List collections
mongosh --quiet --eval 'db.getSiblingDB("invitavideodb").getCollectionNames()'

# Count render records
mongosh --quiet --eval 'db.getSiblingDB("invitavideodb").renders.countDocuments()'

# Print the latest 10 complete documents
mongosh invitavideodb --quiet --eval \
  'db.renders.find().sort({created_at:-1}).limit(10).forEach(printjson)'

# Check backend storage mode
curl -fsS http://127.0.0.1:8001/api/health

# Stop local MongoDB
brew services stop mongodb-community@8.0
```

## Safety checklist

- Back up before bulk updates, deletes, restores, or schema migrations.
- Run a matching `find()` and confirm its result before `updateMany()` or `deleteMany()`.
- Never expose MongoDB port `27017` to the public internet.
- Keep MongoDB bound to `127.0.0.1` when the backend runs on the same server.
- Do not commit database passwords or `.env` files to Git.
- Prefer password prompts over passwords written directly in terminal commands.
- Remember that deleting MongoDB metadata does not delete corresponding media files.
